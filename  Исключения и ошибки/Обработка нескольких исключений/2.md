## Практика #2: `final` в `catch`

В объявлении:

```java
try {
    ...
} catch (final Exception e) {
    ...
}
```

`final` означает, что переменной `e` нельзя присваивать новое значение внутри блока `catch` (она становится «непереназначаемой»):

```java
catch (final Exception e) {
    // e = new Exception(); // ошибка компиляции
}
```

Это **не** делает сам объект исключения «неизменяемым» — это ограничение только на переменную-ссылку `e`.

## Вложенные `try`

Операторы `try` можно вкладывать, например, когда внутреннюю операцию хочется обработать отдельно:

```java
try {
    try {
        riskyInner();
    } catch (IOException e) {
        System.out.println("Обработали внутреннюю I/O-ошибку");
    }
    riskyOuter();
} catch (Exception e) {
    System.out.println("Обработали всё остальное снаружи");
}
```
