## Практика №1 — final-метод нельзя переопределять

Если в суперклассе метод объявлен как `final`, то в подклассе его **нельзя** переопределить — будет ошибка компиляции.

```java
class Parent {
    public final void sayHello() {
        System.out.println("Привет из Parent");
    }
}

class Child extends Parent {
    @Override
    public void sayHello() { // ОШИБКА: нельзя переопределить final-метод
        System.out.println("Привет из Child");
    }
}
```

Что примерно скажет компилятор:
`cannot override sayHello() in Parent; overridden method is final`
