# Практика #1 — `Object.equals()`

В Java `Object` — суперкласс всей иерархии классов. Метод `equals()` можно переопределять в своих классах, чтобы сравнение объектов происходило по смыслу (по значимым полям), а не по ссылке.

## Пример

Файл: `Point.java` — класс точки на плоскости. Две точки считаются равными, если совпадают `x` и `y`. Также переопределён `hashCode()` (обязательная пара к `equals()` для корректной работы в `HashMap`/`HashSet`).

## Соглашения (контракт) для `equals()`

Реализация должна быть:

1. **Рефлексивной**: `a.equals(a)` всегда `true`.
2. **Симметричной**: `a.equals(b)` ⇔ `b.equals(a)`.
3. **Транзитивной**: если `a.equals(b)` и `b.equals(c)`, то `a.equals(c)`.
4. **Консистентной**: пока значимые поля не меняются, результат `equals()` не меняется.
5. **Ненулевой**: `a.equals(null)` всегда `false`.

И обязательно:

6. Если `a.equals(b) == true`, то `a.hashCode() == b.hashCode()` (иначе ломаются хеш-коллекции).

## Запуск

```bash
javac Point.java
java Point
```
