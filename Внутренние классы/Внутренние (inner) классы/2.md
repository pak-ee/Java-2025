# Практика #2 — доступ внутреннего класса к полям/методам внешнего

Да: внутренний (нестатический) класс имеет доступ ко **всем** членам внешнего класса, включая `private` поля и методы. Это не зависит от спецификаторов доступа внешних членов: внутренний код компилируется так, что доступ разрешён.

Спецификатор доступа **самого внутреннего класса** (`public/protected/...`) влияет только на то, *кто может увидеть/создать этот класс снаружи*, но не на его доступ к членам `Outer`.

## Пример (доступ к private-полям и private-методам внешнего класса)

```java
public class Outer {
    private int counter = 0;

    private void incBy(int delta) {
        counter += delta;
    }

    public class Inner {
        public void work() {
            counter++;     // доступ к private-полю Outer
            incBy(10);     // доступ к private-методу Outer
            System.out.println(counter);
        }
    }

    public Inner createInner() {
        return new Inner();
    }
}
```

```java
public class Demo {
    public static void main(String[] args) {
        Outer outer = new Outer();
        Outer.Inner inner = outer.new Inner();
        inner.work();
    }
}
```

## Пример, если имена конфликтуют (ссылка на enclosing object)

```java
public class Outer {
    private String value = "outer";

    public class Inner {
        private String value = "inner";

        public void printBoth() {
            System.out.println(value);        // inner.value
            System.out.println(Outer.this.value); // outer.value (enclosing object)
        }
    }
}
```

## Важное уточнение

Внутренний (нестатический) класс **не может** иметь обычные `static` поля/методы (кроме `static final` констант), потому что он привязан к экземпляру внешнего класса.
