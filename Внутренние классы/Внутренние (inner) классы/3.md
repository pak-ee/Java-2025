# Практика #3 — доступ внешнего класса к полям/методам внутреннего

Да: внешний класс имеет доступ ко **всем** членам своего внутреннего класса, включая `private` поля и методы. Это симметрично: `Outer` ↔ `Inner` внутри одного *top-level* класса могут обращаться к `private` друг друга.

Спецификаторы доступа внутреннего класса (`public/protected/...`) влияют на то, *кто может объявлять переменные этого типа/создавать экземпляры снаружи*, но **внутри** `Outer` доступ к его `Inner` и его членам полный.

## Пример (Outer читает private-поле Inner и вызывает private-метод)

```java
public class Outer {
    public class Inner {
        private int secret = 42;

        private int secretPlus(int x) {
            return secret + x;
        }
    }

    public int useInnerSecrets() {
        Inner i = new Inner();
        int a = i.secret;          // доступ к private полю Inner
        int b = i.secretPlus(8);   // доступ к private методу Inner
        return a + b;
    }
}
```

## Пример, как модификатор Inner влияет на доступ извне

```java
public class Outer {
    private class HiddenInner {
        private String data = "x";
    }

    // Снаружи HiddenInner недоступен как тип,
    // но Outer может создать и использовать его внутри себя.
    public String getHiddenData() {
        HiddenInner h = new HiddenInner();
        return h.data;
    }
}
```

```java
public class Demo {
    void demo() {
        Outer o = new Outer();
        // Outer.HiddenInner x; // ERROR: HiddenInner имеет private-доступ в Outer
        System.out.println(o.getHiddenData()); // OK: используем публичный метод Outer
    }
}
```
