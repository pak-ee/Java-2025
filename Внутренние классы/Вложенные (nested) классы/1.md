## Практика #1 — вложение класса в интерфейс

В Java класс, объявленный внутри `interface`, является **неявно `public static`** (т.е. это *вложенный (nested) класс*, а не внутренний (inner) класс). Поэтому экземпляр такого класса создаётся **без объекта интерфейса**.

### Пример 1: `static`-класс внутри интерфейса и вызов его метода

```java
public interface Utils {
    // Неявно: public static class Printer
    class Printer {
        public void print(String msg) {
            System.out.println(msg);
        }
    }
}

class Demo1 {
    public static void main(String[] args) {
        Utils.Printer printer = new Utils.Printer(); // создание
        printer.print("Hello from nested class");    // вызов метода
    }
}
```

### Пример 2: вложенный класс с `static`-методом (вызов без создания объекта)

```java
public interface MathOps {
    class Calc {
        public static int sum(int a, int b) {
            return a + b;
        }
    }
}

class Demo2 {
    public static void main(String[] args) {
        int result = MathOps.Calc.sum(2, 3); // вызов static-метода
        System.out.println(result);
    }
}
```
