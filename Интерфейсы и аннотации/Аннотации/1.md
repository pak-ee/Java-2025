Практика #1

@Override
Нужно, чтобы показать: “я переопределяю метод”. Если ошибся в названии или параметрах, компилятор ругнётся.
Пример:
class A { void f() {} }
class B extends A {
  @Override
  void f() {}
}

@Deprecated
Помечает метод/класс как устаревший. IDE и компилятор предупреждают, что лучше так не использовать (обычно есть замена).
Пример:
class Api {
  @Deprecated
  void oldMethod() {}
  void newMethod() {}
}

@SuppressWarnings
Отключает некоторые предупреждения компилятора в конкретном месте, если вы уверены, что всё нормально.
Примеры:
@SuppressWarnings("deprecation") // не ругаться на устаревшее
void g(Api api) { api.oldMethod(); }

@SuppressWarnings("unchecked") // не ругаться на unchecked cast
void h() {
  java.util.List list = new java.util.ArrayList();
  java.util.List<String> s = list;
}
